#### 1. **新建文档后文件夹自动关闭 + 文档自动进入重命名状态** ✅

**问题**：在文件夹中新建文档后，文件夹会关闭，新文档不会自动进入重命名状态

**修复方案**：
- **文件夹状态**：修改 `handleCreateChild` 逻辑，确保文件夹保持展开状态而不是切换状态
- **自动重命名**：新增状态管理机制，让新创建的文档自动进入编辑模式

**技术实现**：
```typescript
// 确保文件夹展开（而不是切换）
if (!isExpanded) {
  onToggleExpand(doc.uid);
}

// 新文档自动进入编辑状态
useEffect(() => {
  if (newlyCreatedDocId === doc.uid && doc.type === "document") {
    setIsEditing(true);
    setEditTitle(doc.title);
  }
}, [newlyCreatedDocId, doc.uid, doc.title, doc.type]);
```

#### 2. **保存文档后跳回第一个文档问题** ✅

**问题**：保存任何文档后，激活的文档会自动跳回到第一个打开的文档

**根本原因**：`DocsContext` 中的文档恢复逻辑会在每次 `docs` 数组更新时触发，包括保存文档时，导致 `activeDocId` 被重置为第一个文档

**修复方案**：
- **分离初始化和同步逻辑**：只在应用初始化时从 localStorage 恢复文档状态
- **保护激活状态**：在后续的文档更新中不再改变 `activeDocId`
- **智能同步**：当文档列表更新时，同步打开的文档列表但保持当前激活状态

**技术实现**：
```typescript
// 只在初始化时恢复文档状态
useEffect(() => {
  if (!initialized || docs.length === 0 || docsRestored) return;
  // ... 恢复逻辑，只执行一次
  setDocsRestored(true);
}, [initialized, docs, workspaceType, docsRestored, activeDocId, setActiveDocId]);

// 分离的同步逻辑，不影响 activeDocId
useEffect(() => {
  if (!docsRestored) return;
  // ... 只同步打开的文档列表
}, [docs, openDocs, docsRestored]);
```

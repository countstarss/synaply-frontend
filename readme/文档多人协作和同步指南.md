文档多人协作与同步实现指南

  概述

  本文档介绍如何在 Synaply 项目中实现文档的多人实时协作功能，包括前端与 NestJS 后端的集成方案。

  技术架构

  核心技术栈

  - Y.js: 分布式数据结构，用于实现 CRDT（无冲突复制数据类型）
  - BlockNote: 富文本编辑器，内置 Y.js 支持
  - WebSocket: 实时通信协议
  - IndexedDB: 本地持久化存储
  - Dexie.js: IndexedDB 的封装库

  架构图

  ┌─────────────┐     WebSocket      ┌─────────────┐
  │   前端应用   │ ←────────────────→ │ NestJS 后端  │

  └─────────────┘                    └─────────────┘
         ↓                                   ↓
  ┌─────────────┐                    ┌─────────────┐
  │  IndexedDB  │                    │   数据库     │

  2. WebSocket Gateway
```
  // docs.gateway.ts
  import { WebSocketGateway, WebSocketServer, OnGatewayConnection, OnGatewayDisconnect } from
  '@nestjs/websockets';
  import { Server, Socket } from 'socket.io';
  import * as Y from 'yjs';
  import { setupWSConnection } from 'y-websocket/bin/utils';

  @WebSocketGateway({
    cors: {
      origin: process.env.FRONTEND_URL || 'http://localhost:3000',
      credentials: true,
    },
  })
  export class DocsGateway implements OnGatewayConnection, OnGatewayDisconnect {
    @WebSocketServer()
    server: Server;

    private docs = new Map<string, Y.Doc>();

    handleConnection(client: Socket) {
      console.log(`Client connected: ${client.id}`);

      // 设置 Y.js WebSocket 连接
      setupWSConnection(client, this.docs);
    }

    handleDisconnect(client: Socket) {
      console.log(`Client disconnected: ${client.id}`);
    }
  }
```
  3. 文档服务
``` ts
  // docs.service.ts
  import { Injectable } from '@nestjs/common';
  import { InjectRepository } from '@nestjs/typeorm';
  import { Repository } from 'typeorm';
  import { Document } from './entities/document.entity';

  @Injectable()
  export class DocsService {
    constructor(
      @InjectRepository(Document)
      private docsRepository: Repository<Document>,
    ) {}

    // 创建文档
    async createDocument(createDocDto: CreateDocumentDto) {
      const doc = this.docsRepository.create({
        ...createDocDto,
        createdAt: new Date(),
        updatedAt: new Date(),
      });
      return await this.docsRepository.save(doc);
    }

    // 获取工作区文档列表
    async getDocumentsByWorkspace(workspaceType: string, userId: string) {
      return await this.docsRepository.find({
        where: { workspaceType, userId },
        order: { order: 'ASC' },
      });
    }

    // 更新文档元数据
    async updateDocument(id: string, updateDocDto: UpdateDocumentDto) {
      await this.docsRepository.update(id, {
        ...updateDocDto,
        updatedAt: new Date(),
      });
      return await this.docsRepository.findOne({ where: { id } });
    }

    // 删除文档
    async deleteDocument(id: string) {
      // 递归删除子文档
      const children = await this.docsRepository.find({ where: { parentId: id } });
      for (const child of children) {
        await this.deleteDocument(child.id);
      }
      return await this.docsRepository.delete(id);
    }
  }
```

  5. Y.js 文档持久化
``` ts
  // y-persistence.service.ts
  import { Injectable } from '@nestjs/common';
  import * as Y from 'yjs';
  import { MongoClient, Db } from 'mongodb';

  @Injectable()
  export class YPersistenceService {
    private db: Db;

    constructor() {
      this.initDB();
    }

    private async initDB() {
      const client = await MongoClient.connect(process.env.MONGODB_URL);
      this.db = client.db('synaply-yjs');
    }

    // 保存 Y.js 文档状态
    async saveDoc(docName: string, ydoc: Y.Doc) {
      const collection = this.db.collection('documents');
      const state = Y.encodeStateAsUpdate(ydoc);

      await collection.replaceOne(
        { name: docName },
        { name: docName, data: state, updatedAt: new Date() },
        { upsert: true }
      );
    }

    // 加载 Y.js 文档状态
    async loadDoc(docName: string): Promise<Y.Doc | null> {
      const collection = this.db.collection('documents');
      const doc = await collection.findOne({ name: docName });

      if (doc && doc.data) {
        const ydoc = new Y.Doc();
        Y.applyUpdate(ydoc, doc.data.buffer);
        return ydoc;
      }

      return null;
    }
  }
```

  同步流程

  1. 文档创建流程
``` 
  sequenceDiagram
      participant U as 用户
      participant F as 前端
      participant B as 后端
      participant D as 数据库

      U->>F: 创建新文档
      F->>F: 生成 UUID
      F->>F: 创建本地 Y.Doc
      F->>B: 发送创建请求
      B->>D: 保存文档元数据
      B->>F: 返回文档信息
      F->>F: 更新本地 IndexedDB

  2. 实时协作流程

  sequenceDiagram
      participant U1 as 用户1
      participant U2 as 用户2
      participant WS as WebSocket服务
      participant YJS as Y.js

      U1->>WS: 连接到文档房间
      U2->>WS: 连接到文档房间
      U1->>YJS: 编辑文档
      YJS->>WS: 发送更新
      WS->>U2: 广播更新
      U2->>YJS: 应用更新
```

  权限控制

  文档访问权限
``` ts
  // 在 WebSocket 连接时验证权限
  @UseGuards(WsAuthGuard)
  handleConnection(client: Socket) {
    const user = client.data.user;
    const docId = client.handshake.query.docId;

    // 验证用户是否有权访问该文档
    if (!this.hasAccess(user, docId)) {
      client.disconnect();
      return;
    }

    setupWSConnection(client, this.docs);
  }
```

  性能优化

  1. 延迟保存
``` ts
  // 使用防抖减少保存频率
  const debouncedSave = debounce(async () => {
    await persistenceService.saveDoc(docName, ydoc);
  }, 2000);

  ydoc.on('update', debouncedSave);
```
  2. 增量同步

  Y.js 自动处理增量更新，只传输变化的部分，而不是整个文档。

  3. 离线支持

  故障排查

  常见问题

  1. 连接失败
    - 检查 WebSocket 服务器地址
    - 验证 CORS 配置
    - 检查防火墙设置
  2. 同步延迟
    - 检查网络连接
    - 优化 WebSocket 服务器性能
    - 考虑使用 CDN 加速
  3. 数据丢失
    - 确保 IndexedDB 正确初始化
    - 检查 Y.js 持久化配置
    - 实现自动保存机制

  总结

  通过结合 Y.js、BlockNote 和 WebSocket，我们实现了一个功能完整的多人实时协作文档系统。前端使用 IndexedDB
  提供离线支持，后端通过 NestJS 管理文档元数据和协作会话。这种架构既保证了良好的用户体验，又具有良好的扩展性。
